# CHAT QUICKSTART

## これは何か

`AI-Meeting-Framework` を、Gemini や一般的な AI チャットで使うための簡易導入ファイルです。

この framework は本来、複数ファイルと会議ログ保存を前提にした構造ですが、普通のチャットでも次の形でかなり使えます。

- 会議モードを指定する
- 参加者の役割や性格を指定する
- その場で transcript 形式で話し合わせる
- 必要なら summary や report を出させる

## 普通のチャットでできること

- `character` モードで会議をさせる
- `teaching` モードで記事やエッセイを読ませる
- `Human-in-the-Loop` で自分も会議に参加する
- 複数視点の意見をまとめさせる

## 普通のチャットで難しいこと

- repo 全体を自動で読み込んで厳密運用すること
- `sessions/` へ自動保存すること
- `config/` を横断して常時一貫運用すること

つまり、**単発の会議や読解はやりやすいが、repo ベースの継続運用は弱い** です。

## まず使うなら

次の 3 つのどれかを選ぶと始めやすいです。

### 1. character モード

向いているもの:
- 雑談会議
- 企画会議
- 複数視点での議論
- 見ていて面白い会話

### 2. teaching モード

向いているもの:
- ニュース記事
- インタビュー記事
- 論説
- エッセイ
- 批評文

### 3. live council mode

向いているもの:
- リアルタイム風の会議
- 数ラウンドでの対話
- 人が途中参加する会議

## そのまま貼って使える起動文

### A. character モード用

```md
以下の設定で会議してください。

- モード: character
- 会議形式: Human-in-the-Loop
- 参加者:
  - 司会: 全体整理と結論担当
  - 推進役: 前向きに価値や進め方を考える
  - 現実派: リスクや失敗要因を厳しく見る
  - 分析役: 論点整理と比較を行う
  - 私: 人間参加者

進め方:
1. 司会がテーマを整理する
2. 各役が短く発言する
3. 私が途中で意見を入れる
4. 最後に司会が結論をまとめる

出力形式:
- 会話形式で表示
- 最後に summary を出す

会議テーマ:
（ここにテーマを書く）
```

### B. teaching モード用

```md
以下の設定で Teaching Mode を行ってください。

- teacher: 本質を見抜き、分かりやすく教える役
- questioner: 読者の代わりに質問し、理解を補助する役

目的:
- 対象の文章を、事実、解釈、立場表明、レトリックに分けて読む
- 読み手が誤解しやすい部分を補足する
- 最後に、どのように読むべきかを短くまとめる

出力形式:
1. questioner が疑問を出す
2. teacher が整理して答える
3. 何度か会話する
4. 最後に 5 段階評価と総評を出す

対象の文章:
（ここに記事や文章を貼る）
```

### C. live council mode 用

```md
以下の設定で Live Council Mode を行ってください。

- ラウンド数: 3
- 参加者:
  - 司会
  - 推進役
  - 現実派
  - 分析役
  - 私

ルール:
- 1ラウンドごとに各役が短く話す
- 私はラウンドの間に介入できる
- 最後に結論と次アクションをまとめる

会議テーマ:
（ここにテーマを書く）
```

## 記事を読むときのコツ

### ニュース記事
- 何が確認済み事実かを先に分ける
- 見出しが本文より強くないかを見る

### インタビュー記事
- 発言者の立場を先に確認する
- 発言内容と客観事実を混同しない

### エッセイ、論説
- 面白さと論理の強さを分ける
- 主語が大きくなっていないかを見る
